However, this is not sustainable. P-values are random variables, and hence will likely produce false positives (even when the null hypothesis is guaranteed true) –> introduce procedures, ‘power’ etc. to limit errors.

In procedures, we define parameters for what we could call ‘significant’. Family wise error rates (FWER) determines overall how many results are false negative / false positive.

Aim of procedure is to minimise the FWER: making the probability of making 1< mistakes small. Submitting alpha = 0.05 would always return 5% of gene data as false positive - but 5% of 1000 gene data is still too large.

Therefore FWER determines error rate as probability of making one false positive across the entire dataset. Therefore, the smaller the FWER, the better. If FWER = 0.05, this means 5% probability of producing false positive once in the entire dataset, and vice versa, 95% confidence that the remainder contains zero false positives. This is much more sustainable / conservative.

Multiple testing

Bonferroni correction: controlling the FWER, continues logic above^ To achieve total FWER of 0.05: divide FWER/m, where m = total tests. Conservative - can lead to very high false negative rate. i.e. if dataset = 1000, desired FWER of 0.05 = 0.00005. This means pvalue < 0.00005 to be classed significant.

Sidak’s procedure: minimising alpha (threshold for p-value) based on size of dataset, assuming data are independent (although not applicable to gene investigation, if investigating an interdependent effect)

The above procedures punish any single false positive. In genomics, where genes are not independent, tolerating a small percentage in false positives is sustainable.

False discovery rate (FDR) = false positive (discoveries) / total significant discoveries Benjamini-Hochberg (BH) procedure: controls FDR by ranking p-values (lowest-highest), define threshold (alpha), all p-values until that cut-off are declared significant. If the data had nothing of significance (pure noise), then all p-values would get ‘equally spaced’ p-values, i.e. just as likely to get p-value 0.4 as with p-value 0.9 etc… If the data was of significance, then p-values may cluster (spike) on one end, i.e. p-value 0.03 more likely than 0.08 etc. Therefore, BH procedure loosens the cut-off threshold for each ranked p-value (unlike Bonferonni with strict cut off at alpha (0.05)). This means BH is more lenient to catch the cluster of similar p-values.

Q-values

q-values reflect the FDR. If FDR = FP / (FP + TP), then FDR = proportion of false positives.

If the dataset had nothing of significance (pure noise), then: 1% of p-values are < 0.01; 2% of p-values are < 0.02; 5% of p-values are < 0.05; 95% of p-values are < 0.95 etc. because pure noise would have equally distributed p-values. Each p-value is equally as likely to occur as any other (hence ‘equally spaced’ / uniform).

This means we can predict the expected false positive rate (noise classed as false positive). Say in a dataset of 1000 genes, rank 40 gene has p-value = 0.002. (Referencing the above p-values for pure noise): if the dataset had nothing of significance (pure noise), then a p-value of 0.002 would mean we expected 0.2% of data to be positive. 0.002 * 1000 = 2 genes expected to be positive. However, because FDR ranks p-values, all ranks above gene rank 40 are also p value < 0.002. This means that 40 p-values (genes) were positive (significant < 0.002). Therefore, there may be some false positives. FDR = (expected) FP / (FP + TP) = 2 / 40 = 0.05 (5%) = q-value.

p-value examines probability of false positive on an independent basis. q-value examines probability of false positive on the said rank based on all ranks above (group), using expected FP.
